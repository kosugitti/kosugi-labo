ラボノートの書き方

ラボノートとは

ラボノートは、ゼミ活動の記録と先生とのコミュニケーションツールです。皆さんのGitHubリポジトリに Labo-note.Rmd というファイルが入っています。

毎週のゼミの前に、その週の記録を書いてコミット・プッシュしてください。先生はこれを読んで、皆さんの状況を把握します。

書くこと

毎回の記録は 報告・連絡・相談 の3つで構成されます。

報告

今週やったことを書きます。研究に限らず、Rドリル・読んだ論文・考えたこと、なんでも構いません。

「何もしなかった」も立派な報告です。正直に書いてください。何もしなかったことを隠す方が問題です。

ヒント報告の例
  • Rチュートリアル U02のCフェイズを完了、U02のBフェイズ 2-B-3 まで進んだ
  • 卒論テーマ候補として「大学生の睡眠と学業成績」について調べた
  • 先週のゼミで紹介された論文（田中, 2020）を読んだ。方法のサンプルサイズが小さいのが気になった
  • 今週は体調不良で何もできなかった

連絡

教員への事務的な連絡事項を書きます。口頭やDiscordで伝えた内容でも、記録として残しておくと後で便利です。

ヒント連絡の例
  • 来週のゼミは就活の面接のため欠席します
  • 推薦書をお願いしたいです（○○大学院、締切: 7/15）
  • 倫理審査の書類を確認してほしいです
  • 学会発表のポスター印刷を研究室のプリンタでやっていいですか
  • 特になし

相談

困っていること、迷っていること、聞きたいことを自由に書いてください。研究に関係ないことでも構いません。雑談でもOKです。

研究の相談は、何に困っているかを具体的に書いてくれると、ゼミの時間を有効に使えます。

ヒント相談の例
  • 卒論テーマが2つあって決められない。A案は面白そうだけどデータが取りにくい
  • ggplotで凡例の位置を変える方法がわからない
  • read_csvでエラーが出る。ファイルパスの指定が合っているか自信がない
  • 大学院に進学するか就職するか迷っている
  • 「○○○○」というテキストを研究室で買ってほしい
  • M実験準備室にモニタを追加してほしい
  • 最近キーボードを買い替えたいが、おすすめはあるか
  • 特になし

書き方の手順

1. ファイルを開く

自分のリポジトリにある Labo-note.Rmd を RStudio や VSCode で開きます。

2. テンプレートをコピーする

ファイルの一番下にあるテンプレートを、<!-- ↓↓↓ ここから上にコピー ↓↓↓ --> の上にコピーします。

## 2026-04-16

### 報告

- Rチュートリアル U01のCフェイズまで完了した
- U02のBフェイズに取り組み中

### 連絡

- 特になし

### 相談

- read_csvでエラーが出る。ファイルパスが合っているか自信がない

---

3. 日付を書き換えて内容を記入する

2026-XX-XX をその日の日付に変えて、各セクションに記入します。

4. コミット・プッシュする

git add Labo-note.Rmd
git commit -m "ラボノート 2026-04-16"
git push

RStudio の Git タブからでもOKです。

よくある質問

毎週書かないといけないの？

はい。ゼミがある週は必ず書いてください。長期休暇中は月1回程度で構いません。

何も進捗がないときはどうすればいい？

「報告: 今週は何もできなかった」と正直に書いてください。理由があれば書いてもらえると助かりますが、なくても構いません。書かないよりも「何もなかった」と書く方がずっと良いです。

先生はいつ読むの？

ゼミの前に目を通します。相談や連絡事項があれば、ゼミの中で対応します。急ぎの連絡はDiscordを使ってください。

Rmdで書く理由は？

将来的に、コードの実行結果や図表を埋め込んだ報告ができるようになるためです。最初のうちはMarkdown部分だけ使えば大丈夫です。
